# EXPRESS

프로젝트 디렉토리 구조는 다음과 같습니다.

```jsx
|-src (소스폴더)
|	|-config (설정폴더)
|	|-controller (MVC => Controller)
|	|-model (MVC => Model)
|	|-legacy (old)
|	|-route (라우팅)
|-Swagger (스웨거 세팅)
```

package.json 에 서버 실행 스크립트가 작성 되어있습니다.

```json
{
  "name": "kisti_server",
  "version": "1.0.0",
  "description": "",
  "main": "server.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "dev": " node -r dotenv/config src/server.js dotenv_config_path=.env.dev",
    "start": "node -r dotenv/config src/server.js dotenv_config_path=.env.prod"
  },
  "author": "",
  "license": "ISC",
  "dependencies": {
    "agora-token": "^2.0.3",
    "body-parser": "^1.20.2",
    "cors": "^2.8.5",
    "dotenv": "^16.4.5",
    "express": "^4.19.2",
    "mysql2": "^3.10.2",
    "nodemon": "^3.1.4",
    "swagger-jsdoc": "^6.2.8",
    "swagger-ui-express": "^5.0.1",
    "uuid": "^10.0.0",
    "ws": "^8.18.0"
  }
}

```

scripts 항목에 dev 가 서버 실행 코드이며 .env.dev 가 DEV 파일로 세팅됩니다.

```
SERVER = DEV
USE_APIPORT = 8081
USE_WSPORT = 8082
```

해당 하는 ENV 파일을 열어보시면 세팅을 바꿀 수 있습니다.

서버 실행 방법 

서버 프로젝트가 있는 곳으로 powershell을 이동후 npm run dev 를 실행 시키면 됩니다.

```powershell
PS E:\Work\KISTi\KISTi_Server> npm run dev

> kisti_server@1.0.0 dev
> node -r dotenv/config src/server.js dotenv_config_path=.env.dev

DEV API SERVER Running on 8081
WS Server Running on 8082

```

Docker에서는 forever 모듈로 설치하면 됩니다.
