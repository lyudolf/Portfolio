- Currently, users automatically enter the "TestGroup" room in Photon.
- The VR cognitive behavior project is fixed to login 20 and pwd 20 (constant!!!!!).
- User info will be changed to the new server API on Amazon EC2 with Docker.
- The room name will be set to be the same for both Agora and Photon (testgroup's hash???).

## Agora Controller and Photon Controller Modularization

- Currently, there are PhotonManager and AgoraManager.
- PhotonManager contains flow control functions for connecting to Photon and accessing rooms.
- It is advisable to avoid directly controlling other managers from PhotonManager and instead adopt a callback format.

- AgoraManager should be separated from Photon. There are also parts that control other managers here.

- The callback format will be changed.
- It would be good to group Agora and Photon under the concept of sessions and distribute session events with EventListeners.

## Server API Improvement Plan

- The current server is configured with PHP pages based on phpMyAdmin for the API.
- The API is sent from WebRequest_Kaist, and results are received to directly control the managers.

- The part that calls the API and controls the data should be separated.
- Instead of PHP pages, NodeJs Express will be introduced.
- Amazon EC2 and Docker will be utilized.
