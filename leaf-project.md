## **1. 게임 컨셉 및 핵심 루프**

### **1.1 기본 컨셉**

플레이어는 넓은 정원의 관리인이 되어 가을에 떨어진 낙엽들을 정리합니다. 단순히 낙엽을 줍는 것에서 시작하여, 도구를 해금하고, 점점 더 어려운 환경에서 효율적으로 작업하는 것이 목표입니다.

**왜 이런 컨셉인가?**

- 정리/청소 게임은 본능적인 만족감을 제공합니다
- 단순한 조작으로 누구나 즉시 플레이 가능
- 시각적 피드백이 명확함 (낙엽이 줄어드는 것이 눈에 보임)
- 스트레스 해소 + 성취감의 조합

### **1.2 핵심 게임 루프**

```
[낙엽 수집] → [100개 모으면 봉투 생성] → [봉투를 쓰레기통에 투척] → [돈 획득] → [업그레이드 구매] → [다음 스테이지 진행]

```

**루프 설계 의도:**

1. **즉각적 보상**: 낙엽을 주울 때마다 숫자가 올라감 → 즉각적 피드백
2. **중간 단계 보상**: 100개 모으면 봉투 생성 → 작은 성취감
3. **최종 보상**: 봉투 판매 → 돈 획득 → 구매력 상승
4. **장기 목표**: 스테이지 클리어 → 새로운 도구/콘텐츠 해금

### **1.3 조작 방식**

| **입력** | **동작** |
| --- | --- |
| WASD | 이동 |
| 마우스 | 시점 회전 (1인칭) |
| Shift | 달리기 |
| Space | 점프 |
| 좌클릭 | 도구 사용 / 봉투 들기 |
| 우클릭 | 봉투 던지기 |
| 1~4 | 도구 변경 |
| E | 게이트 상호작용 / 튜토리얼 닫기 |
| Tab | 상점 열기 |
| H | 도움말 |

**1인칭 시점을 선택한 이유:**

- 몰입감 극대화
- 정리 작업의 "내가 직접 하는" 느낌
- 도구 사용 시 손에 들린 도구가 보여서 현실감 증가
- VR 확장 가능성 고려

## **2. 스테이지 시스템**

### **2.1 스테이지 구조**

게임은 총 5개의 스테이지로 구성됩니다. 각 스테이지는 물리적으로 연결된 하나의 거대한 마당이며, 울타리로 구역이 나뉘어 있습니다.

| **스테이지** | **목표 낙엽 수** | **해금 도구** | **특수 기믹** | **난이도** |
| --- | --- | --- | --- | --- |
| Stage 1 | 500개 | 손 (기본) | 없음 | ★☆☆☆☆ |
| Stage 2 | 1200개 | 갈퀴 | 두더지 출현 | ★★☆☆☆ |
| Stage 3 | 2500개 | 송풍기 | 거대 두더지 | ★★★☆☆ |
| Stage 4 | 2000개 | - | 거대 두더 | ★★★★☆ |
| Stage 5 | 1800개 | - | 천둥, 스나이퍼 몰, 토네이도 | ★★★★★ |

### **2.2 스테이지 진행 방식**

**게이트 시스템:**

- 각 스테이지 경계에는 울타리와 게이트가 있음
- 현재 스테이지의 목표를 달성하면 "게이트 오픈 가능" 상태가 됨
- 플레이어가 게이트 앞에서 `E` 키를 눌러야 다음 스테이지 오픈

**왜 자동 오픈이 아닌가?**

- 플레이어에게 "준비됐을 때 진행" 선택권 부여
- 이전 스테이지에서 돈을 더 모으고 싶을 수 있음
- 다음 스테이지 튜토리얼을 보기 전 정리할 시간 제공
- 명확한 진행 의식(Ritual)이 성취감 증대

### **2.3 스테이지별 설계 의도**

**Stage 1 - 튜토리얼 영역**

- 목표: 게임의 기본 조작 학습
- 장애물 없음, 넓고 평평한 공간
- 200개라는 적은 목표로 빠른 성취감
- 집 1채 + 울타리만 배치

**Stage 2 - 첫 번째 도전**

- 목표: 도구 시스템 학습 + 첫 번째 방해 요소 도입
- 갈퀴 해금으로 효율 증가 체험
- 두더지가 땅에서 튀어나와 낙엽을 흩뜨림
- "정리해도 다시 흩어진다" → 긴장감 조성

**Stage 3 - 송풍기의 필요성**

- 목표: 새로운 도구의 전략적 활용
- 낙엽이 800개로 크게 증가
- 거대 두더지 등장 (더 넓은 범위 영향)
- 송풍기로 낙엽을 한 곳에 모으는 전략 필요

**Stage 4 - 환경 변수 대응**

- 목표: 예측 불가능한 상황 대처
- 바람 장치(Air Vent)가 낙엽을 날려버림
- 장치 위치를 파악하고 우회하거나, 역이용하는 전략

**Stage 5 - 최종 도전**

- 목표: 모든 기술을 총동원한 최고 난이도
- 천둥: 랜덤 위치에 벼락이 치며 낙엽 폭발
- 스나이퍼 몰: AI가 플레이어 전방을 예측하여 공격
- 토네이도: 봉투를 날려버림
- 1800개의 압도적인 물량

## **3. 도구 시스템**

### **3.1 도구 종류 및 특성**

### **🖐️ 손 (HAND)**

- **해금 조건**: 기본 제공
- **동작**: 좌클릭 홀드 시 주기적으로 낙엽 수집
- **범위**: 반경 2m
- **수집량**: 1개 (업그레이드 시 최대 100개)
- **수집 간격**: 0.2초

**설계 의도:**

- 가장 기본적이고 직관적인 도구
- "손으로 직접 줍는다"는 현실감
- 느리지만 정확한 수집
- 업그레이드 여지가 많음 (수집량 증가)

### **🪥 갈퀴 (RAKE)**

- **해금 조건**: Stage 2 클리어
- **동작**: 좌클릭 홀드 시 전방 낙엽을 끌어모음
- **범위**: 반경 3m (업그레이드 가능)
- **효과**: 낙엽에 힘을 가해 플레이어 방향으로 이동

**설계 의도:**

- 손보다 넓은 범위 처리
- "끌어모은다"는 현실적 동작
- 수집이 아닌 "이동"이므로 봉투 만들기 전 정리용
- 송풍기와 조합하여 효율 극대화

### **🌬️ 송풍기 (BLOWER)**

- **해금 조건**: Stage 3 클리어
- **동작**: 좌클릭 홀드 시 전방에 바람 발사
- **범위**: 반경 4m, 최대 거리 8m (업그레이드 가능)
- **효과**: 낙엽을 강하게 밀어냄

**설계 의도:**

- 갈퀴와 반대 개념 (끌어오기 vs 밀어내기)
- 대량의 낙엽을 빠르게 이동
- "한 곳에 몰아놓고 손으로 수집"하는 전략
- 시각적으로 화려한 효과 (낙엽이 날아가는 모습)

### **🌀 진공청소기 (VACUUM) - 히든 기능**

- **해금 조건**: 키보드 4번으로만 접근 가능 (UI 없음)
- **동작**: 전방의 낙엽을 빨아들여 즉시 수집
- **범위**: 반경 5m

**설계 의도:**

- 테스트/디버그용 강력한 도구
- 정식 출시 시 유료 DLC 또는 게임 클리어 후 해금 예정
- 현재는 숨겨진 이스터에그

### **3.2 도구 애니메이션**

모든 도구는 1인칭 시점에서 손에 들린 상태로 보입니다.

- **손**: 클릭 시 "움켜쥐는" 펄스 애니메이션 (0.2초 주기)
- **갈퀴**: 끌어당기는 모션
- **송풍기**: 미세한 흔들림 + 바람 파티클 효과

**왜 도구가 보여야 하는가?**

- 현재 장착 도구를 직관적으로 인지
- 동작 피드백 제공 (클릭했는데 반응이 있음)
- 게임의 현실감 증대

## **4. 낙엽 물리 시스템**

### **4.1 기술적 결정의 기획적 이유**

**8,000개의 낙엽을 선택한 이유:**

- 시각적 밀도감: "정말 많은 낙엽이 있다"는 인상
- 청소의 보람: 줄어드는 것이 확연히 보임
- 도전 요소: 많은 양을 효율적으로 처리해야 함

**InstancedMesh 기반 렌더링:**

- 8,000개 개별 오브젝트는 성능상 불가능
- 하나의 메시로 통합하여 최적화
- 플레이어 입장에서는 차이 없음 (보이는 건 동일)

### **4.2 낙엽 물리 규칙**

| **상태** | **동작** |
| --- | --- |
| 공중 | 중력 적용, 자연스럽게 낙하 |
| 바닥 | 마찰력 적용, 천천히 멈춤 |
| 충돌 | 집/나무/울타리에 부딪히면 튕김 |
| 수집됨 | Y좌표 -1000으로 이동 (화면에서 사라짐) |

**바닥에서 평평하게 눕히는 이유:**

- 시각적 자연스러움: 실제 낙엽은 바닥에 평평하게 쌓임
- 수집 용이성: 세로로 서 있으면 클릭하기 어려움
- 회전 감쇠 0.5 적용하여 빠르게 눕힘

### **4.3 구역 제한 시스템**

**울타리 충돌:**

- 외곽 울타리: X=-15~~135, Z=-12~~12
- 스테이지 구분 울타리: X=15, 45, 75, 105
- 천장 제한: Y=20m (낙엽이 하늘로 날아가는 것 방지)

**CCD(Continuous Collision Detection) 적용:**

- 문제: 빠르게 날아가는 낙엽이 울타리를 뚫고 지나감
- 해결: "이전 위치"와 "현재 위치"를 비교하여 선을 넘었는지 판정
- 넘었다면 원래 위치로 되돌림 + 반발력 적용

**왜 낙엽은 게이트를 통과하지 못하는가?**

- 게이트는 "플레이어 전용"
- 각 스테이지의 낙엽은 해당 스테이지에서만 존재해야 함
- 스테이지 진행도 계산의 정확성 보장
- "다른 스테이지 낙엽이 섞이면 안 됨"

## **5. 환경 장애물 시스템**

### **5.1 두더지 (Mole)**

**등장 스테이지**: Stage 2~4

**동작 패턴:**

1. 땅 속에서 대기 (보이지 않음)
2. 3~5초 간격으로 솟아오름
3. 2초간 지상에서 대기
4. 다시 땅 속으로 들어감

**낙엽 영향:**

- 두더지와 충돌한 낙엽은 위로 튀어오름
- 정리해둔 낙엽 더미를 흩뜨림

**설계 의도:**

- 단순한 수집 작업에 긴장감 추가
- "방해 요소"의 첫 등장
- 플레이어가 두더지 위치를 기억하고 피해가는 전략 유도

### **5.2 바람 장치 (Air Vent)**

**등장 스테이지**: Stage 4~5

**동작:**

- 특정 위치에서 위쪽으로 강한 바람 발생
- 범위 내 낙엽을 공중으로 날려버림
- 플레이어도 밀려남

**설계 의도:**

- "구역 제한" 개념 도입
- 플레이어가 지역을 우회하거나 빠르게 통과해야 함
- 송풍기와 조합하여 역이용 가능 (바람 방향으로 낙엽 밀기)

### **5.3 천둥 (Thunder)**

**등장 스테이지**: Stage 5

**동작 패턴:**

1. 10초마다 천둥 버스트 발생
2. 0.2초 간격으로 5회 연속 낙뢰
3. 각 낙뢰는 Stage 5 내 랜덤 위치
4. 낙뢰 반경 5m 내 낙엽 폭발적으로 흩어짐

**시각 효과:**

- 번개 기둥 렌더링
- 지면 충격파
- 글로벌 플래시 (화면 전체 밝아짐)

**설계 의도:**

- 예측 불가능한 카오스 요소
- "방금 정리했는데!" 하는 좌절감 → 도전 의욕
- Stage 5의 "재난" 테마 강화

### **5.4 스나이퍼 몰 (Mole Sniper)**

**등장 스테이지**: Stage 5

**동작 패턴:**

1. AI가 플레이어의 위치와 시선 방향 분석
2. 낙엽 밀도 맵 구축 (어디에 낙엽이 많은지)
3. 플레이어 전방 + 낙엽 밀집 지역을 겨냥
4. 10초마다 해당 위치에 폭발 발생
5. 폭발 시 "료(Ryo)" 캐릭터가 나타났다가 사라짐

**AI 예측 로직:**

- ONNX 모델 사용 (선택적)
- 폴백: 휴리스틱 기반 타겟팅

**설계 의도:**

- "지능적인 적"의 등장
- 플레이어를 방해하는 것이 목적임을 명확히 함
- 정리하려는 곳을 먼저 공격 → 심리전
- 캐릭터 등장으로 유머 요소 추가

### **5.5 토네이도 (Tornado)**

**등장 스테이지**: Stage 5

**동작:**

- Stage 5 내에서 랜덤하게 이동
- 접촉한 봉투를 하늘로 날려버림
- 낙엽에는 영향 없음

**설계 의도:**

- 봉투 시스템에 대한 직접적 위협
- "100개 모아서 봉투 만들었는데 날아감!" → 극적 긴장감
- 토네이도 위치를 피해 안전한 곳에서 봉투 생성 전략

## **6. 경제 시스템**

### **6.1 재화 흐름**

```
낙엽 수집 → 점수(Score) 증가
100점 → 봉투 생성 (점수 -100)
봉투 판매 → 돈(Money) 150원 획득
돈 → 업그레이드 구매

```

### **6.2 업그레이드 시스템**

| **업그레이드** | **효과** | **가격** |
| --- | --- | --- |
| 수집량 Lv.2 | 한 번에 3개 수집 | 300원 |
| 수집량 Lv.3 | 한 번에 5개 수집 | 700원 |
| 수집량 Lv.4 | 한 번에 10개 수집 | 1,500원 |
| 수집량 Lv.5 | 한 번에 50개 수집 | 5,000원 |
| 수집량 Lv.6 | 한 번에 100개 수집 | 10,000원 |
| 돈 배율 x2 | 봉투 판매 시 2배 | 2,000원 |
| 돈 배율 x3 | 봉투 판매 시 3배 | 5,000원 |

## **8. 성능 최적화 결정**

### **8.1 낙엽 시스템 최적화**

**문제**: 8,000개의 낙엽 물리 계산이 매 프레임 발생

**해결책:**

1. **거리 계산 최적화**
    - `distanceTo()` 대신 `distanceToSquared()` 사용
    - 제곱근 계산 생략으로 CPU 절약
2. **분산 충돌 검사**
    - 무거운 충돌 검사(집, 나무)를 3프레임에 나눠서 실행
    - 각 낙엽이 3프레임마다 한 번씩만 충돌 검사
3. **수면(Sleep) 시스템**
    - 바닥에 정지한 낙엽은 물리 계산 스킵
    - 속도가 거의 0이면 "수면" 상태로 전환
    - 외부 힘(바람, 도구)이 가해지면 "깨어남"
4. **리스폰 로직 제거**
    - 원래: 맵 밖으로 나간 낙엽을 다시 맵 안으로 리스폰
    - 변경: 완전 제거 (성능 우선)
    - 대신 울타리 충돌을 강화하여 애초에 나가지 않도록

**기획적 의도:**

- 저사양 기기에서도 플레이 가능해야 함
- 60fps 유지가 게임 경험에 중요
- 물리 정확도보다 성능 우선

### **8.2 렌더링 최적화**

- InstancedMesh로 8,000개를 1개 드로우콜로 처리
- 그림자는 선택적 적용
- LOD(Level of Detail) 없음 (낙엽이 너무 작아서 불필요)

## **10. AI 시스템 개발 상세**

### **10.1 스나이퍼 몰 (MoleSniper) - ONNX 딥러닝 모델**

### **학습 데이터 및 방식**

**입력 데이터 (324차원):**

```
[낙엽 밀도 맵 20×16=320] + [플레이어 X,Z 정규화 2] + [플레이어 시선 방향 2]

```

**밀도 맵 구축 방식:**

1. Stage 5 영역(X: 105~~135, Z: -12~~12)을 20×16 그리드로 분할
2. 각 셀 크기: 1.5m × 1.5m
3. 각 셀에 존재하는 낙엽 수 카운트
4. 최대값으로 정규화 (0~1 범위)

**학습 목표:**

- 플레이어가 **작업하려는 방향**을 예측
- 그 방향의 **낙엽이 밀집된 영역**을 타겟팅
- 플레이어를 최대한 방해하는 위치 선정

**모델 구조:**

```
Input (324) → Dense (512, ReLU) → Dense (320) → Softmax

```

- 출력: 320개 셀에 대한 확률 분포
- ArgMax로 최고 확률 셀 선택

**제약 조건 (Validation):**

| **조건** | **값** | **이유** |
| --- | --- | --- |
| 최소 거리 | 1.0m | 너무 가까우면 플레이어가 피할 수 없음 |
| 최대 거리 | 3.0m | 너무 멀면 방해 효과 감소 |
| 시야각 제한 | ±55° | 플레이어 전방 원뿔 내에서만 타겟팅 |

**Fallback 로직:** ONNX 모델 로드 실패 시 휴리스틱 기반 폴백:

```
score=가로채기점수+(밀도 ×0.2)

```

- 가로채기 점수: 플레이어 진행 방향과의 정렬도
- 밀도: 해당 셀의 낙엽 수

**설계 의도:**

- 단순 랜덤이 아닌 **플레이어 행동 예측 기반** 공격
- "내가 가려는 곳을 알고 있다"는 느낌 → 긴장감
- 딥러닝 모델이므로 예측 불가능한 패턴

### **10.3 AI 도우미 (Potato/Carrot/Tomato) - 유한 상태 머신**

### **상태 머신 구조**

```
IDLE ↔ MOVING_TO_BAG ↔ CARRYING_BAG

```

| **상태** | **동작** |
| --- | --- |
| IDLE | 가장 가까운 봉투 탐색 → 발견 시 이동 시작 |
| MOVING_TO_BAG | 목표 봉투로 이동 → 1.5m 이내 도달 시 픽업 |
| CARRYING_BAG | 가장 가까운 쓰레기통으로 이동 → 3.0m 이내 도달 시 판매 |

### **봉투 탐색 알고리즘**

```
functionfindNearestBag(){
letnearest=null;
letminDist=Infinity;

for (각봉투) {
constdist=내위치.distanceTo(봉투위치);
if (dist<minDist) {
minDist=dist;
nearest=봉투;
        }
    }
returnnearest;
}

```

### **이동 로직**

```
// 목표 방향 계산
constdirection=목표위치.sub(내위치).normalize();

// 속도 적용 (4 m/s)
api.velocity.set(direction.x*4,-1,direction.z*4);

// 목표를 바라보도록 회전
api.rotation.set(0,Math.atan2(direction.x,direction.z),0);

```

**물리 설정:**

- 질량: 50
- 고정 회전: true (넘어지지 않음)
- 충돌 그룹: AI 전용 (플레이어와 충돌하지 않음)

### **AI 도우미별 차별점**

| **도우미** | **모델** | **이동 속도** | **해금 비용** |
| --- | --- | --- | --- |
| 감자 (Potato) | Potato_SH.glb | 4 m/s | 기본 |
| 당근 (Carrot) | Carrot.glb | 5 m/s | 중급 |
| 토마토 (Tomato) | Tomato.glb | 6 m/s | 고급 |

**설계 의도:**

- 플레이어는 **수집**에 집중, AI는 **운반** 담당
- 분업을 통한 효율 극대화
- 업그레이드를 통한 점진적 자동화
